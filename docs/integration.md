# Integration

## 概要

このドキュメントは、`Garoon_pre` Android アプリと `Garoon_pre_api` ローカル API サーバーをローカル環境で連携して動作確認するための手順をまとめたものです。

- アプリ本体: `Garoon_pre`
- ローカル API: `Garoon_pre_api`

本プロジェクトでは Android アプリ側を主題とし、API サーバーは動作確認用の最小構成として扱います。

---

## 前提

以下が利用できることを前提とします。

- Android Studio
- Android エミュレーター、または USB 接続した Android 実機
- Node.js
- npm
- `adb`
- `Garoon_pre` と `Garoon_pre_api` の両リポジトリを clone 済みであること

---

## リポジトリ構成

### Garoon_pre

Android アプリ本体です。

主に以下を担当します。

- UI
- 状態管理
- ローカルキャッシュ
- 同期処理
- テスト
- アーキテクチャ設計

### Garoon_pre_api

ローカルでの動作確認用 API サーバーです。

主に以下を担当します。

- Android アプリから利用する API の提供
- ローカル開発時の疎通確認
- デモ、検証用バックエンド

本番運用向けの完全なバックエンドではなく、開発・確認用途を目的とした最小構成です。

---

## 連携の全体像

ローカル開発時の接続イメージは次の通りです。

### Android エミュレーター利用時

- API サーバーはホスト PC 上で起動する
- Android エミュレーターからは `10.0.2.2` を使ってホスト PC の localhost にアクセスする

接続先 URL:

```text
http://10.0.2.2:3000/
```

### USB 接続した実機利用時

- API サーバーはホスト PC 上で起動する
- `adb reverse` を使って端末からホスト PC の 3000 番ポートへ接続できるようにする

接続先 URL:

```text
http://localhost:3000/
```

---

## 1. API サーバーを起動する

`Garoon_pre_api` ディレクトリへ移動して、以下を実行します。

```bash
npm install
npm start
```

起動後のベース URL は以下です。

```text
http://localhost:3000
```

必要に応じて、ヘルスチェックエンドポイントで起動確認を行ってください。

```text
GET /health
```

ブラウザまたは `curl` で確認する例:

```bash
curl http://localhost:3000/health
```

---

## 2. Android アプリから接続する

### 2-1. エミュレーターで動かす場合

Android エミュレーターからホスト PC 上の API サーバーへ接続する場合は、以下を使用します。

```text
http://10.0.2.2:3000/
```

`10.0.2.2` は Android エミュレーターから見たホスト PC の localhost を表します。

### 2-2. 実機で動かす場合

USB 接続した Android 実機から API サーバーへ接続する場合は、先に以下を実行します。

```bash
adb reverse tcp:3000 tcp:3000
```

この設定により、端末の `localhost:3000` へのアクセスがホスト PC の `localhost:3000` に転送されます。

そのうえで、アプリ側の接続先 URL は以下を使用します。

```text
http://localhost:3000/
```

---

## 3. 動作確認の手順

基本的な確認手順は以下の通りです。

1. `Garoon_pre_api` を起動する
2. Android アプリを起動する
3. アプリ内で API の接続先 URL を設定する
4. ログイン画面から接続確認を行う
5. スケジュール、掲示板などの主要画面でデータ取得を確認する

---

## 4. 接続先 URL の使い分け

開発時は、実行環境に応じて接続先を切り替えてください。

| 実行環境 | 接続先 |
| --- | --- |
| Android エミュレーター | `http://10.0.2.2:3000/` |
| USB 接続した実機 | `http://localhost:3000/` |

---

## 5. トラブルシュート

### API サーバーに接続できない

以下を順に確認してください。

- `Garoon_pre_api` が起動しているか
- `http://localhost:3000/health` にホスト PC からアクセスできるか
- エミュレーター利用時に `10.0.2.2` を使っているか
- 実機利用時に `adb reverse tcp:3000 tcp:3000` を実行しているか
- ポート番号が `3000` になっているか

### 実機でだけ接続できない

以下を確認してください。

- USB デバッグが有効になっているか
- `adb devices` で端末が認識されているか
- `adb reverse tcp:3000 tcp:3000` を再実行したか

確認コマンド:

```bash
adb devices
adb reverse tcp:3000 tcp:3000
```

### ログインできない

以下を確認してください。

- API 側のユーザー情報や seed データが正しいか
- アプリ側の接続先 URL が正しいか
- API サーバー起動直後にエラーが出ていないか

### 一部画面だけデータ取得に失敗する

以下を確認してください。

- API 側で対象エンドポイントが実装されているか
- アプリ側のリクエスト先パスが正しいか
- API サーバーのログにエラーが出ていないか

## 6. 開発時の運用方針

連携手順の正本はこのドキュメントに集約します。

- アプリと API の連携方法は `Garoon_pre/docs/integration.md` を正本とする
- `Garoon_pre_api` 側 README には API 単体の起動方法を記載する
- リポジトリをまたぐ手順は、このドキュメントを参照する

これにより、接続方法や確認手順の重複管理を避けます。
